# イベントストーム

イベントストームは、複雑なビジネスプロセスの探索とドメイン駆動設計（DDD）における境界コンテキストのモデリングに使用される協働ワークショップ技法です。これは、イベントとその相互作用を大型のホワイトボードやデジタルプラットフォーム上で視覚化することにより、システムを通じてこれらのイベントが時間とともにどのように流れているかを集中する技術です。この技法は、領域の理解、潜在的な問題の特定、そしてチームがビジネスプロセスについての理解を調整するのに役立ちます。

## イベントストームとは何か？

イベントストームは、チームが領域とその内蔵のイベントの流れを理解するための協働ワークショップです。イベントとその相互作用を大型のホワイトボードやデジタルプラットフォーム上で視覚化し、これらのイベントがシステムを通じて時間とともにどのように流れているかを集中する技術です。

## キー・フィーチャー

1. **協働アプローチ**: 開発者、プロダクトオーナー、領域専門家など、異なる役割の参加者が一緒に領域をマッピングします。
2. **イベントの焦点**: 技術は、イベントの流れとそのシステムへの影響を理解することに重点を置いています。
3. **視覚的表現**: イベント、エンティティ、境界は単純なグラフィックスを使用してシステムの視覚マップを作成します。
4. **タイムトラベリング**: 参加者はシステムがどのように時間とともに進化するかを想像し、過去、現在、未来の異なるステートとイベントを視覚化します。
5. **領域マッピング**: ドメインをマッピングして、ビジネスプロセスについてのチームの理解を深め、調整します。

## 歴史

イベントストームは、2012年にソフトウェア開発の会議でグレゴル・ホホペによって初めて導入されました。この技法は、複雑なビジネスプロセスとシステム間の相互作用を明らかにする効果性から、ソフトウェア開発コミュニティで大きな支持を得ました。「イベントストーム」という名前は、ビジネス領域内でのイベントの嵐を描くというアイデアから来ています。

## 使用例

1. **ドメイン解析**: 変化するビジネス領域を理解するため、イベントの流れを分解することで役立ちます。
2. **モデリング**: イベント駆動モデルの作成を助けることで、ソフトウェアシステムの設計に利用できます。
3. **要件集約**: システムの異なる部分がどのように相互作用するかを視覚化することで、要件を収集します。
4. **アーキテクチャ設計**: イベントがシステムを通じてどのように流れているかをマッピングすることで、イベント駆動アーキテクチャを設計します。
5. **チーム連携**: チームメンバー間の協力を向上させるため、システムについての共有理解を提供します。

## インストール

イベントストームには特定のソフトウェアをインストールする必要はありませんが、次のツールと材料は役立ちます：

- **大型ホワイトボードやフリップチャート**: イベントの流れを視覚化します。
- **マーカーとポストイット**: イベントとエンティティをラベル付けします。
- **デジタルツール**: MiroやMuralなどのツールは、リモートイベントストームに有用です。

## 基本的な使用方法

1. **準備**: 開発者、プロダクトオーナー、領域専門家などの異なる役割の参加者を集めます。
2. **導入**: イベントストームの概念とセッションの目標を簡単に説明します。
3. **領域マッピング**: シンプルなグラフィックスを使用してエンティティ、イベント、境界をマッピングします。
4. **イベントマッピング**: 最初のイベントから始めて、そのシステムへの影響を追跡しながらイベントの流れをマッピングします。
5. **タイムトラベリング**: システムがどのように時間とともに進化するかを話し合います。異なるステートとイベントを考慮します。
6. **ディスカッションと改善**: モデルを精密にし、すべてのチームメンバーが共有理解を持てるように話し合います。
7. **ドキュメンテーション**: 見つかった結果を記録し、システムの開発に使用します。

### 例

たとえば、小売ビジネス領域では、顧客が注文を提出し、商品が発送され、支払いが行われるなどがあります。イベントストームのプロセスでは、「注文が提出された」、「注文が発送された」、「支払いが受け取られた」というイベントとそれらが「顧客」、「注文」、「在庫」などのエンティティとの相互作用をマッピングします。

これらのイベントとその影響を視覚化することで、チームは領域を深く理解し、システム内の潜在的な瓶颈や効率性の問題を特定することができます。

## 結論

イベントストームは、複雑なシステムを理解し、チームが領域についての共有理解を調整するための強力な技法です。イベントとその相互作用を集中することで、効果的かつ効率的なソフトウェアシステムの設計に役立ちます。ドメイン解析、要件集約、またはアーキテクチャ設計のために使用される場合でも、イベントストームは協働かつ視覚的なアプローチを提供します。
