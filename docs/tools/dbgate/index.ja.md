# DBGate

DBGateは、オープンソース（MIT）のウェブベースデータベース管理ツールで、phpMyAdmin、Adminer、DBeaver、DataGripといった従来のツールに代わるモダンな選択肢です。Node.js/ExpressバックエンドとReactフロントエンドで構築されており、すべてウェブブラウザ上で動作するクリーンで洗練されたユーザーインターフェースを提供するため、クロスプラットフォーム対応で、クラウド、サーバー、コンテナ環境に最適です。

## DBGateを選ぶ理由

従来のデータベースクライアントは、オペレーティングシステムへのインストールが必要な場合が多く、チームや環境間で断片化を引き起こします。DBGateは完全にブラウザベースであるため、以下のことが可能です。

- **データベースをリモートで管理**：SSHトンネルやネイティブクライアントが不要。
- **Dockerスタックに統合**：開発データベースへ即座にアクセス。
- **接続とスクリプトを共有**：認証付きの集中インスタンスを通じて。
- **同一のウェブインターフェースでWindows、macOS、Linux間をシームレスに操作**。

## 主な機能

| 機能                       | 説明                                                                                          |
|----------------------------|-----------------------------------------------------------------------------------------------|
| 複数データベース対応        | MySQL、MariaDB、PostgreSQL、SQL Server、MongoDB、SQLite、CockroachDB、Amazon Redshift、Redisに同時接続可能。 |
| 高度なSQLエディタ          | シンタックスハイライト、インテリジェントな自動補完、マルチタブクエリ、充実したクエリ履歴。              |
| スキーマ/データブラウザ    | データベースオブジェクトの参照、作成、変更、削除。強力なソート・フィルタ機能によるインラインデータ編集。 |
| ER図                      | エンティティ・リレーションシップ図を自動生成し、データベーススキーマを可視化。                      |
| エクスポート/インポート    | CSV、JSON、SQL、Markdown、Excelへのエクスポート、CSVおよびSQLファイルからのインポート。         |
| 外部キーナビゲーション    | データブラウザから直接、関連レコードへドリルダウン。                                             |
| サーバーモニタリング      | アクティブプロセス、サーバーステータス、変数設定の表示。                                          |
| Docker最適化              | どのサーバーにも簡単にデプロイできる公式Dockerイメージ。                                          |
| デスクトップアプリ        | Windows、macOS、Linuxでスタンドアロン利用可能なElectronバンドル版。                               |

## インストール

DBGateは複数の方法でインストール・実行できます。

### 1. Docker（サーバー推奨）

```bash
docker run -d -p 3000:3000 --name dbgate dbgate/dbgate
```

その後 `http://localhost:3000` にアクセスしてください。

`docker-compose.yml` を使用する場合：

```yaml
version: '3'
services:
  dbgate:
    image: dbgate/dbgate
    ports:
      - "3000:3000"
    restart: unless-stopped
```

### 2. Node Package Manager (NPM)

```bash
npm install -g dbgate
dbgate
```

アクセス： `http://localhost:3000`

### 3. デスクトップインストーラー

Windows、macOS、Linux向けのビルド済みインストーラーは [GitHub releases page](https://github.com/dbgate/dbgate/releases) からダウンロードできます。

### 4. クラウドデプロイ

Heroku、Railwayなどのプラットフォーム向けにワンクリックデプロイが利用可能です。

## クイックスタート / 使い方

### 1. DBGateを起動

ブラウザで `http://localhost:3000` にアクセスします。

### 2. 接続を追加

**接続**の横にある **+** アイコンをクリックします。データベースエンジン（例：PostgreSQL）を選択し、ホスト、ポート、ユーザー名、パスワード、データベースなどの接続情報を入力します。

### 3. データを参照

保存した接続をクリックすると、データベース/テーブルのツリーが表示されます。テーブルをクリックして行を表示します。

### 4. データベースにクエリを実行

**クエリ**ボタンをクリックしてSQLエディタを開きます。SQLを記述し、**実行**（または `Ctrl+Enter`）を押します。

### 5. スキーマを可視化

データベースまたはテーブルを右クリックし、**ER図**を選択するとビジュアルスキーマが生成されます。

### 6. データをエクスポート

テーブルまたは結果セットを右クリックし、**エクスポート**を選択すると、希望の形式（CSV、JSON、SQLなど）でデータをダウンロードできます。

## コマンド例

**DockerでDBGateを起動し、データを永続化する：**

```bash
docker run -d \
  -p 3000:3000 \
  -v dbgate-data:/home/app/.dbgate \
  --name dbgate \
  dbgate/dbgate
```

**開発スタック内のローカルPostgreSQLインスタンスと一緒に使用する：**

```yaml
# docker-compose.yml
version: '3.8'
services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_PASSWORD: example
  dbgate:
    image: dbgate/dbgate
    ports:
      - "3000:3000"
    depends_on:
      - postgres
```

**npmでインストールして実行する：**

```bash
npm install -g dbgate
dbgate
```

**環境変数を使用して接続する（高度な使い方）：**

```bash
docker run -d \
  -e DBGATE_SERVER_NAME=myPostgres \
  -e DBGATE_SERVER_TYPE=postgres \
  -e DBGATE_SERVER_HOST=192.168.1.100 \
  -e DBGATE_SERVER_PORT=5432 \
  -e DBGATE_SERVER_USER=admin \
  -e DBGATE_SERVER_PASSWORD=secret \
  -p 3000:3000 \
  dbgate/dbgate
```

## ユースケース

1. **リモートサーバー管理** – SSHトンネリングやネイティブクライアントをインストールせずに、VPSやクラウドインスタンス上のデータベースを管理。
2. **開発環境** – 開発者にローカルデータベースへのGUIアクセスを即座に提供するため、`docker-compose.yml` スタックにDBGateを含める。
3. **チームツール** – 認証を適切に設定した集中型DBGateインスタンスをデプロイし、チームで開発・ステージングデータベースへのアクセスを共有。
4. **教育・トレーニング** – クライアントインストールを管理することなく、学生にSQLインターフェースを迅速に提供。
5. **クロスプラットフォームワークフロー** – 同じウェブインターフェースを使って、オペレーティングシステム間をシームレスに切り替え。

## アーキテクチャ

DBGateは以下の要素で構成されます。

- **バックエンド：** Node.js/Expressサーバーがデータベース接続、クエリ実行、APIエンドポイントを処理。
- **フロントエンド：** ReactベースのSPAが、SQLエディタ、データブラウザ、スキーマビューアを含むユーザーインターフェースを提供。
- **データベースドライバ：** ネイティブNode.jsドライバーまたはODBC/JDBCブリッジを介して複数のデータベースエンジンをサポート。

アプリケーションは接続情報、SQLスクリプト、その他のオブジェクトをローカルストレージ（またはホスト版の場合はオプションのクラウドストレージ）に保存します。Dockerイメージはシングルポートデプロイのためにすべての依存関係をバンドルしています。

## 制限事項

- **高度なIDE機能：** IntelliJ DataGripにあるような一部の機能（分散リファクタリング、高度なコード解析など）が不足する場合があります。
- **パフォーマンス：** ブラウザで非常に大きなデータセット（10万行以上）をレンダリングする場合、ネイティブアプリよりも遅くなることがあります。エクスポート操作はサーバーサイドで処理されるため、パフォーマンスは向上します。
- **認証：** オープンソース版には組み込みのユーザー認証はありません。チームで使用する場合は、リバースプロキシ（nginx + auth_basicなど）を前面に配置する必要があります。

## まとめ

DBGateは、軽量ウェブクライアント（phpMyAdminなど）と高機能ネイティブIDE（DataGripなど）のギャップを埋める、パワフルで柔軟なオープンソースデータベース管理ツールです。クロスプラットフォーム対応、コンテナフレンドリーな設計、そして拡大を続ける機能セットにより、モダンなウェブネイティブデータベースクライアントを求める開発者、DBA、チームにとって優れた選択肢です。
