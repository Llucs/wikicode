# 使用 Rust 创建一个真正的命令行工具

## 概述

"使用 Rust 创建一个真正的命令行工具" 项目是一个全面的指南和实际练习，帮助你通过构建一个真正的命令行接口 (CLI) 工具来学习 Rust。这个指南旨在帮助开发者理解语言的语法及其生态系统，包括 Rust 的标准库和流行的各种库。该项目旨在提供一个动手的学习体验，涵盖了模块化设计、错误处理、配置管理以及测试等主题。

## 核心功能

1. **模块化设计**：工具被分解成更小、更易于管理的模块。
2. **可定制和扩展**：用户可以通过添加新功能或修改现有功能来扩展工具。
3. **错误处理**：强大的错误处理机制，确保工具可靠且用户友好。
4. **配置管理**：支持配置文件和命令行参数。
5. **文档**：全面的文档，指导用户完成开发过程。
6. **测试**：包含单元测试和集成测试，确保代码质量并保持可维护性。

## 安装

### 先决条件

1. **安装 Rust**：确保你已经安装了 Rust。你可以遵循官方的 Rust 安装指南来设置你的环境。
2. **安装 Cargo**：Cargo 是 Rust 包管理器，当你安装 Rust 时会自动安装。

### 安装项目的步骤

1. **克隆仓库**：从 GitHub 克隆 "Create-a-Real-World-CLI-Tool-in-Rust" 仓库。
   ```sh
   git clone https://github.com/rust-lang-nursery/create-a-cli-tool.git
   ```

2. **构建项目**：导航到项目目录，并使用 Cargo 构建工具。
   ```sh
   cd create-a-cli-tool
   cargo build --release
   ```

3. **运行工具**：使用 Cargo 生成的二进制文件运行工具。
   ```sh
   cargo run
   ```

## 基本用法

1. **运行工具**：从命令行执行工具。
   ```sh
   cargo run
   ```

2. **查看帮助**：大多数 CLI 工具包含一个帮助菜单，可以通过 `--help` 标志访问。
   ```sh
   cargo run -- --help
   ```

3. **自定义行为**：使用命令行参数和配置文件来定制工具的行为。

4. **与工具交互**：根据工具的功能，你可以输入数据、指定文件路径或配置设置等。

## 示例用法

对于一个假设的工具 `file-manipulator`，基本用法可能如下：

```sh
# 列出目录中的所有文件
cargo run -- list /path/to/directory

# 重命名文件
cargo run -- rename old_filename new_filename

# 删除文件
cargo run -- delete /path/to/file
```
