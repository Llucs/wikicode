# Saga パターン

##  Overview

Saga パターンは、複数のサービスやリソース間でのトランザクションを管理するためのデザインパターンです。このパターンは、トランザクションが有効とされるまでに完了する一連の操作を維持し、トランザクションが一致していることを確実にします。もし操作が失敗した場合、パターンは完了した操作の全てを回復してシステムの整合性を保つことができます。

## キー フEATURES

1. **補償操作**: 各単位の作業（操作）に対して、その変更を元の状態に戻すための補償操作が定義されています。操作が失敗した場合、システムは以前の状態に戻ることができます。
2. **順序的な実行**: 操作は特定の順序で実行され、各操作は前の操作の成功に依存します。
3. **最終的な一貫性**: パターンは、個々のトランザクションが失敗してもシステムが時間と共に一貫した状態に近づくことを保証します。
4. **一貫性**: サゲア内の操作は、同じ操作を複数回呼び出したとしてもシステムの状態が変わらないように一貫した操作であるべきです。

## 歴史

Saga パターンは、マイクロサービスアーキテクチャにおける分布型トランザクションを管理するための課題に対処するために開発されました。マイクロサービスが普及する以前には、モニolith アプリケーションはデータベースレベルでトランザクションを管理していました。しかし、アプリケーションがより分散化したことで、複数のサービス間でのトランザクションの管理の複雑さが増しました。Saga パターンは、これらの複雑さを扱うための解法として導入されました。

Saga の概念は、1970年代にジェム グレイによってトランザクション処理の作業で形成されましたが、マイクロサービスと分散システムの文脈で注目されるようになったのは2010年代です。

## 使用例

1. **金融トランザクション**: 取引（送金、支払い、返金）を処理するときには、口座間で正確に資金が移動するかを確認する必要があります。Saga はこれらの操作を管理し、送金が失敗した場合に元の残高を回復することができます。
2. **オーダー処理**: エコマースでは、オーダーの処理には商品予約の作成、在庫の更新、顧客の請求など、複数のステップが必要です。Saga は、これらの操作が全て成功するか失敗した場合に全ての操作を完了または回復することができます。
3. **医療システム**: 医療では、請求、診察予約、処方箋管理などのトランザクションは、全てのステップが完了または失敗した場合に回復することで患者データの一貫性を維持する必要があります。
4. **保険請求**: 保険請求は、請求処理、支払い、文書検証などのステップから構成されています。Saga はこれらの操作を管理し、プロセスの一部が失敗した場合にも請求が適切に処理されたり回復したりします。

## インストールと設定

Saga パターンは、アプリケーション プログラミングとミドルウェア サービスの組み合わせを使用して通常は実装されます。以下は、Saga を設定する基本的な手順です。

1. **操作の定義**: Saga の一部として実行する必要がある操作を特定します。各操作に対して補償アクションを定義します。
2. **メッセージ キューの使用**: 操作の実行を管理するためのメッセージ キューを実装します。RabbitMQ, Kafka, AWS SQS などのメッセージ ブローカーがこれに該当します。
3. **Saga マネージャ**: 操作の実行、Saga のステートの追跡、操作が失敗した場合の補償ロジックの管理を行うSaga マネージャを作成します。
4. **補償アクション**: 操作が失敗した場合にシステムを以前の状態に戻すための補償アクションを実装します。

### 基本的な使用方法

1. **Saga の開始**: Saga を開始するためにシーケンスの最初の操作を起動します。
2. **操作の実行**: シーケンスの操作を順番に実行します。操作が失敗した場合、Saga は補償アクションを実行し停止します。
3. **ステートの追跡**: Saga の進行状況を追跡するためにステートフルなレコードを維持します。操作が正しい順序で完了することを確保します。
4. **補償**: 操作が失敗した場合、Saga はシステムを一貫した状態に戻すための補償アクションを実行します。
5. **Saga の完了**: すべての操作が成功して完了した場合、Saga は完了としてマークされます。

### サンプル

以下は、Python でSaga の基本構造を示すサンプルコードです。操作はキューに登録され、シーケンスで実行され、操作が失敗した場合に補償アクションを定義します。

```python
from queue import Queue

# 操作と補償アクションの定義
def create_product_reservation(product_id, quantity):
    # 商品予約の作成を実装します
    pass

def update_inventory(product_id, quantity):
    # 在庫の更新を実装します
    pass

def charge_customer(customer_id, amount):
    # 顧客の請求を実装します
    pass

def cancel_reservation(product_id, quantity):
    # 商品予約のキャンセルを実装します
    pass

def refund_customer(customer_id, amount):
    # 顧客の返金を実装します
    pass

# Saga の定義
def process_order(saga_id, product_id, quantity, customer_id, amount):
    saga_queue = Queue()

    try:
        saga_queue.put(create_product_reservation(product_id, quantity))
        saga_queue.put(update_inventory(product_id, quantity))
        saga_queue.put(charge_customer(customer_id, amount))
        
        while not saga_queue.empty():
            operation = saga_queue.get()
            operation()
        
        # Saga を完了としてマークします
        print(f"Saga {saga_id} は成功しました。")
    except Exception as e:
        # 補償アクションの実行
        while not saga_queue.empty():
            operation = saga_queue.get()
            operation()
        print(f"Saga {saga_id} は失敗しました。補償アクションが実行されました。")
        
# Saga を開始する
process_order(1, "P123", 10, "C12345", 100)
```

このサンプルは、操作はキューに登録され、順番に実行され、操作が失敗した場合に補償アクションを定義する基本的な構造を示しています。

## 結論

Saga パターンは、分散システムにおける複数のサービス間のトランザクションを管理するための堅固なソリューションです。操作が特定の順序で実行され、失敗した場合の補償アクションを提供することで、システムの一貫性を保証します。Saga パターンを理解することは、信頼性とスケーラビリティのあるマイクロサービスアーキテクチャを開発する上で重要です。
