# Wikiを検索

WikiCodeは完全に検索可能です。検索インデックスはデプロイ時に構築され、ブラウザ上で完全に動作するため、クエリは即座に実行され、データがマシンの外部に出ることはありません。

## 検索方法

<div class="grid cards" markdown>

- :material-magnify: __検索バー__

    任意のページの右上にある虫眼鏡アイコンをクリックするか、キーボードの++slash++を押すと、検索モーダルが開きます。

- :material-keyboard: __キーボードショートカット__

    - ++slash++ — 検索バーにフォーカス
    - ++esc++ — 検索モーダルを閉じる
    - ++arrow-up++ / ++arrow-down++ — 結果を移動
    - ++enter++ — ハイライトされた結果を開く

- :material-format-letter-case: __ヒント__

    - 検索は**部分文字列ベース**です。`mkdocs`と入力すると、"mkdocs"を含むページがすべて一致します。
    - デフォルトでは**大文字小文字を区別しません**。
    - 引用符で囲んだフレーズは完全な部分文字列と一致します: `"open hands"`。
    - 複数の単語は、すべての単語を含むページと一致します。

</div>

## インデックスの対象

検索インデックスは、サイト上にレンダリングされたすべてのMarkdownページを対象とします：

- `docs/` 以下の記事、ガイド、リファレンスページ。
- `projects/` 以下のプロジェクト概要。
- `snippets/` 以下のスニペット説明。
- `docs/tools/` 以下のツールページ。
- `blog/` 以下のブログ投稿。
- コードブロックのテキスト（関数名やCLIフラグを検索可能）。

インデックスは `main` へのプッシュのたびに再生成されるため、公開コンテンツと常に同期しています。

## クライアントサイドである理由

- **プライバシー。** クエリがリモートサービスに送信されることはありません。
- **速度。** 入力すると結果が表示されます。
- **コスト。** 静的サイト以外にホストするものはありません。
- **オフライン。** 一度サイトが読み込まれれば、インデックスはブラウザキャッシュに残り、ネットワークなしでも動作し続けます。

## カスタム検索ショートカットの追加

検索モーダルを事前入力した状態で開くディープリンクが必要な場合は、検索が一度フォーカスされた後にサイトURLに `?q=<query>` を追加します。正確な動作はMaterialテーマのバージョンに依存します。推奨される方法は、キーボードショートカット (++slash++) を使用してクエリを入力することです。
